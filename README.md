# How to Curate 2–3 Patterns into Assets

This is the exact, minimal process to turn extracted patterns into usable system knowledge.

---

# Step 1 — Select 2–3 Patterns

From your latest extraction:

- Choose patterns that feel:
  - reusable  
  - decision-relevant  
  - clearly defined  

Do NOT overthink this.

Good signal:
- “I would reuse this”
- “This solves a recurring problem”

---

# Step 2 — Evaluate Each Pattern

For each selected pattern, ask:

### Is this NEW?
- Not already captured in your system  
→ proceed as new  

### Is this a REFINEMENT?
- Same idea as an existing concept, but better  
→ update existing canonical  

### Is this a DUPLICATE?
- Already exists with equal or better clarity  
→ discard or ignore  

---

# Step 3 — Create an Asset

If NEW or REFINED, create a clean asset.

## Asset Format (simple)

### Name
- Use the pattern name (refine if needed for clarity)

### When to Use
- Keep it short and clear

### Action
- Must be directly executable
- No vague language

### Why It Works
- Keep focused on reasoning

---

# Example

### Deterministic Execution Gate

**When to Use**
- When user intent is explicit and inputs are complete  

**Action**
- Detect intent  
- Validate inputs  
- Execute immediately without clarification  

**Why It Works**
- Eliminates conversational drift and ensures consistent behavior  

---

# Step 4 — Assign Status

Decide:

- **Canonical**
  - core, reusable concept  
  - likely to be referenced often  

- **Supporting**
  - useful, but not core  
  - example or edge case  

- **Archive**
  - not worth keeping  

---

# Step 5 — Assign Domain

Choose ONE:

- AI Systems  
- UX  
- Finance  
- etc.  

Do NOT assign multiple.

---

# Step 6 — Tag (lightweight)

- Domain  
- Type → Pattern  
- Concept (1–3 max)  
- Priority (High / Medium / Low)  
- Status  

---

- process entire document  

👉 Just do 2–3 patterns and stop
